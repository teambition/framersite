### Animations Framer supports many different types of animations, including springs and bezier curves.
Property Description Examples
layer layer that is animated
properties properties being animated (x, y, width, height, opacity, scale, rotation, filters)
curve curve used for animation:
  • "linear"
  • "bezier-curve"
  • "spring-rk4" Runge-Kutta spring
  • "spring-dho" Damping harmonic oscillator spring
  • "spring" same as "spring-rk4"
curveOptions options for used curve
  • bezier-curve: "ease-in", "ease-out", "ease-in-out", [0, 1, 0, 1]
  • spring-rk4: friction, tension, velocity, tolerance
  • spring-dho: stiffness, damping, mass, velocity, tolerance
 1
time duration of the animation in seconds (ignored for springs)
delay amount of seconds to delay the animation before start
repeat number of times to repeat animation
start() start the animation
stop() stop the animation
reverse() returns the opposite animation to this one
on(event, function) listen to events: "start", "stop", "end"

### DeviceView The **DeviceView** class emulates a device like an iPhone, iPad or Android. It allows you to scale the device, scale the content and adjust the orientation. After the device is set up, everything will render within its screen. Device previews match mirrored previews, meaning that if you're previewing on iPhone and then view it on an iPhone it will appear in fullscreen. Scaling and orientation properties are also prefixed with Framer.device: 
Framer.Device.contentScale = 0.5
Framer.Device.orientation = 90
Please note that the scale-to-fit view can render your content off-pixel to retain your screen ratio. Your content will always snap-to-pixel when viewed at 50%, 100% etc. If your browser supports it, device images are compressed with JPEG2000 to reduce image size.

Options
